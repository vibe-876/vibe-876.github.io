Programming

Why do I adore programming?

To me, programming is a way of expressing mathematical logic using a notation that a machine can understand -- it lets me apply maths to the real world with very little effort. There are a few langugages that I actively use, which are listed below. There are of course, other languages that I know (for example, the HTML and CSS that I have written this website in), but I choose not to include them in the list of languages, as I use them for a single purpose (in this case, this website) and nothing else.

All of my programming is done using GNU Emacs, which is the best editor that I have found, and I would recommend it to anyone who enjoys tinkering with the tools that they use (although I'll admit, emacs-lisp takes a bit of getting used to).

The languages that I use

In no particular order:

  • Haskell

    • Haskell is a beautiful language, there is no denying that. It seems strange to start with, but it (alongside other languages of the same style, such as the ML family) exposes you to a new way of thinking of data-structures algorithms, and I've found this to be a more natural way for my own mind to comprehend these abstract concepts.

    I mentioned above that I view progrmaming as a form of maths, and Haskell very much suits that. It's not everyone's cup of tea, but it is certainly mine.

    As every Haskell programming does, I use GHC as my compiler, and as many do, I use cabal as my build system. This is because I've found that both of them work quite well inside of Emacs.

  • Rust

    • Rust is a language that I've come to quite like. It seems like a nice middle-ground between Haskell and C -- it lets me do low level programming, with a functional flavour.

  • Lisp(s)

    • Emacs lisp is the one that I use the most, common lisp is the one that I usually use for projects, scheme is the prettiest.

  • C

    • C's elegance -- and hence, C++'s inelegance -- comes from its natural mapping towards machine language. It makes you think in a way that most other languages don't. Despite this, I've stopped writting new projects in it...

    Don't get me wrong, I love C, but frankly, I think that for what I do, Haskell, Lisp, and Rust do the job better. This is not to say that any of the languages is better than C -- I still feel that C has a very firm place in software development, in the deep dark pits of embedded software, when you need every bit and cycle, it is unparalleled. But for what I do, the others serve me better. Despite this, it still has a very firm place in my heart, and I often still play with it; the control that it gives is a lot of fun.
