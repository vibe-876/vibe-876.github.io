Languages

I use a few languages:

  • LISP

    • LISP is a strange one...

    It has no visible limit to how far it can abstract a problem, it has some of the best programming literature to date using about it (namely SICP), and it really does change the way that you think.

    I would recommend that people give it a shot. Once it hits you, it hits you hard, and other languages can start to look weak in comparison. If nothing else, it's worth learning a bit of Scheme just for the sake of reading SICP.

    I've recently been using a lot of common lisp for a random parser that I'm working on, which itself is practise for trying to write a basic dating sim engine (also in common lisp).

  • Haskell

    • I like Haskell because it feels very close to maths, and I can reason about my programs as such. I find it's type system incredible to use, and I'm yet to find another language that's type system compares. Currying allows for some really cool tricks, and I sometimes find myself surprised by how many ways it can be utilised.

    Overall, Haskell can be a pain to learn if you're used to other languages, but it's not impossible, and I think that it's defiantly worth it.

  • C

    • A lot of people seem to think of C as some god-language, a paragon of good design, and that all languages should be like it (no garbage collectors, minimal, etc). Whilst I agree to some extent, I hold a different opinion. To me, C is a high-level assembly language. I can look at some C code, and have a (semi) confident idea on how the assembly will look, which is important for some things that I enjoy, such as systems programming.

    I've heard that Rust deprecates C, although I'm yet to give it a real shot. And if I do learn Rust, I have a feeling that my opinion would not change (I will however say that it very well may deprecate C++, as that's already pretty damn far from assembly). C is good at being a high-level assembly language, and only really goes wrong when people forget that.

  • Scala

    • I've been playing with Scala a lot recently; this is for a 2 main reasons: I like how it handles OOP, and it runs on the JVM.

    Running on the JVM is an obvious advantage, as it allows me to work with many more languages (ie anyone who programs in Java or another JVM language), and allows me to mod Minecraft (which should probably not be a criteria for my chosen programming languages).

    The handling of OOP is less obvious, but I feel that it quickly become apparent when using it. I won't go into detail, but I feel that it blends OOP and FP in a very efficient manner. It will probably go on to replace Python as my OOP language and as my programming with friends language (I just need to convince my friends to learn some JVM language -- probably Java -- first).

  • Python

    • Python has a distinct advantage over all of the languages shown above: my friends know it.

    I tend to use Python more when I want to work with others, as it is such a common language that most programmers know. I don't really think that it does anything special, although I think that it does a decent job at merging different programming styles.

    That being said, it can be a real pain sometimes...
